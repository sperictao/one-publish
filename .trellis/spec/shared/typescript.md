# TypeScript Guidelines

## Generated Tauri Contracts

Use `src/generated/tauri-contracts.ts` for payloads that cross the Rust/TypeScript boundary. The file is generated by `src-tauri/examples/generate_tauri_contracts.rs` through `src-tauri/src/contracts.rs`.

Do not edit generated contracts manually. Run:

```bash
pnpm check:contracts
```

or:

```bash
pnpm generate:contracts
```

## Type Imports

Use `import type` for type-only imports.

```typescript
import type { PublishSpec } from "@/generated/tauri-contracts";
```

## Frontend Wrapper Types

Command wrapper modules should import generated Rust contract types and convert to frontend store types only at the boundary.

Reference files:

- `src/lib/store/api.ts`
- `src/features/publish/publishRuntime.ts`
- `src/lib/store/types.ts`

## Avoid

- Do not duplicate Rust payload types by hand in components.
- Do not hide unknown data behind `any`.
- Do not use non-null assertions to silence contract drift.
- Do not put conversion logic in UI components when `src/lib/store/types.ts` already owns normalization.
