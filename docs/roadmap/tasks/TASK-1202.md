## Context

当前 `src-tauri/src/commands/publish/preflight/path_validation.rs` 的 `looks_like_windows_path` 把任何含 `\` 的字符串都视为 Windows 路径，在 POSIX 上一律降级为 `WindowsStylePathOnPosix`：

- `\\nas01\releases\app`（合法 UNC）在 macOS / Linux 上被判为不合法
- 未来 `sftp://user@host/path` 中如含 `\` 会触发同一误判

`preflight/path_validation.rs:73-86` 的逻辑已无法覆盖远程目录场景。本任务在不破坏现有 Windows / POSIX 校验的前提下，把"UNC 与 URL scheme"从误判清单里摘出。

## Goals

- UNC 路径在所有平台都判为 Compatible
- URL scheme 字符串（已由 TASK-1201 识别为 Remote）跳过传统路径分隔符校验
- 现有"POSIX 上写 `C:\foo`"、"Windows 上写 `/foo`"、"Windows 盘符根不存在"三类原有告警保持不变

## Non-Goals

- 真实的远程可达性 / 鉴权探针（Phase 13）
- 修改 `evaluate_publish_output_access` 的探针策略（拆到 TASK-1203）

## User Stories

- 作为发布到 Windows NAS 的开发者，我希望在 macOS 上配置 `\\nas01\releases\app` 时 preflight 直接判为兼容，而不是先把我拦下来。

## Requirements

### Functional

- R1: `evaluate_publish_output_validation` 在校验前先用 TASK-1201 的 `parse_output_target` 把目标分类，命中 `MountedRemote::Unc` 或 `Remote` 时返回 `Compatible`，不再走 `detect_output_path_validation_issue`。
- R2: 保留对纯 Windows 风格本地路径在 POSIX 上的告警（C:\..., 不带 UNC 头的反斜杠）。
- R3: `looks_like_windows_path` 拆出新的 `looks_like_unc_path` 以避免把 UNC 错配进 Windows-style 误判。
- R4: `PublishOutputValidationIssue` 不新增枚举值（保持 IPC 类型稳定）；UNC 直接通过即可。

### Non-Functional

- N1: 改动只在 `preflight/path_validation.rs` 与同模块单测，不影响 `access.rs`。
- N2: 现有 41 个 path_validation 相关单测全部保持通过。

## UX Notes

- 用户输入 UNC 后，PublishConfigPanel 不再显示"路径不兼容"红条；徽章渲染交给 TASK-1204。

## Technical Design

- 改动文件：
  - `src-tauri/src/commands/publish/preflight/path_validation.rs`
  - `src-tauri/src/commands/publish/preflight/mod.rs`（引入 `parse_output_target`）
- 关键改动：
  ```rust
  pub(super) fn evaluate_publish_output_validation(
      configured_output_dir: Option<&str>,
      inferred_output_dir: &str,
  ) -> PublishOutputValidation {
      let path_to_check = configured_output_dir.unwrap_or(inferred_output_dir).trim();
      if path_to_check.is_empty() { return PublishOutputValidation::not_applicable(); }

      match parse_output_target(path_to_check) {
          OutputTarget::MountedRemote { kind: MountKind::Unc, .. } => compatible(),
          OutputTarget::Remote(_) => compatible(),
          _ => existing_logic(path_to_check),
      }
  }
  ```
- `looks_like_windows_path` 调整：UNC（`\\` 开头）不再触发"Windows-style on POSIX"。
