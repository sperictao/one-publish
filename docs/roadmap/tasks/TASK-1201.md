## Context

发布配置的 `output` 目前是一个裸字符串，preflight / execution 都按"本地路径"处理。当用户配置的输出目录是 **UNC**（`\\server\share\path`）或 **已挂载的远程卷**（macOS `/Volumes/share`、Linux `/mnt/...`、`/media/...`），现状代码：

- `src-tauri/src/commands/publish/preflight/path_validation.rs` 会把含 `\` 的字符串在 POSIX 上误判为 `WindowsStylePathOnPosix`；
- macOS 端 `preflight/access.rs` 的保护目录探针对 `/Volumes/...` 不适用，可能产生不必要的写入探针；
- 前端 `src/lib/paths.ts` 已经有 `UNC_ROOT_RE`，但仅用于 `isRootPath`，没有归一化输出目录类型。

本任务统一所有"目录类型识别"逻辑到一个解析器，作为后续 Phase 13+（远程协议）唯一的入口。

## Goals

- 在 Rust 后端引入统一的 `OutputTarget` 解析器，输出枚举 `Local | MountedRemote | Remote(RemoteUri)`
- 解析顺序固定为：URL scheme → UNC → 平台白名单挂载点 → Local
- 解析结果暴露为 IPC 命令，供前端和 preflight 共用
- 保持 `PublishSpec.parameters.output` 字符串不变，向后兼容旧 spec

## Non-Goals

- 远程协议（SFTP/S3/WebDAV）的真实读写——留到 Phase 13（R1）
- 凭据管理 / Stronghold 接入——留到 Phase 13
- 前端 UI 徽章渲染——拆到 TASK-1204

## User Stories

- 作为开发者，我希望把 `\\nas01\releases\app` 填进输出目录后，preflight 不再误报路径不合法，这样可以直接发到 NAS。
- 作为 macOS 用户，我希望把 `/Volumes/build-share/publish` 设为输出目录后，应用能识别它是挂载远程卷并跳过桌面保护目录写入探针。

## Requirements

### Functional

- R1: 新增 `src-tauri/src/output_target/mod.rs`，导出 `OutputTarget`、`MountKind`、`RemoteUri`、`parse_output_target(raw: &str) -> OutputTarget`。
- R2: `parse_output_target` 解析优先级：
  1. 匹配 `^[a-zA-Z][a-zA-Z0-9+.\-]*://` 视为 `Remote(RemoteUri)`（Phase 12 仅做识别，不调度任何 IO）。
  2. `^\\\\[^\\/]+[\\/][^\\/]+` 视为 `MountedRemote { kind: Unc, .. }`。
  3. 平台白名单前缀：
     - macOS：以 `/Volumes/` 开头；
     - Linux：以 `/mnt/`、`/media/`、`/run/user/<uid>/gvfs/` 开头；
     - Windows：除 UNC 外，盘符走 Local。
     在前缀命中后调用 `statfs`/`GetVolumeInformation` 获取 `fs_type`，命中 `{cifs, smb, smbfs, nfs, nfs4, fuse.sshfs, afpfs, webdav, vboxsf}` 之一即视为 `MountedRemote { kind: Mounted, fs_type }`；否则回落 Local。
  4. 否则 `Local(PathBuf)`。
- R3: 新增 Tauri 命令 `parse_output_target_command(raw: String) -> OutputTargetDescriptor`，用 `ts-rs` 同步导出 TypeScript 类型。
- R4: `OutputTargetDescriptor` 是 `OutputTarget` 的可序列化版本（去除 `PathBuf`，用 `String`），字段全部 `camelCase`，对齐现有 preflight 类型风格。
- R5: 在 `src-tauri/src/lib.rs` 注册新命令；权限放在 `src-tauri/capabilities/default.json` 的 `core:default` 之外（不需要新权限，IPC 内部命令）。

### Non-Functional

- N1: 跨平台行为以条件编译 + 测试覆盖兼顾，主线 `cargo test` 在 Linux CI 必须通过。
- N2: 错误可恢复：scheme 中含非法字符时降级为 Local，不抛错。
- N3: 不引入任何网络 IO；`statfs` 仅在挂载点候选命中后调用一次，需在 50ms 超时下返回（IO 异常时降级为 Local）。
- N4: 解析过程不读 spec，纯字符串输入，便于单测。

## UX Notes

- 不直接影响 UI 的渲染（徽章交给 TASK-1204）。
- 解析结果通过 IPC 暴露后，前端 `PublishConfigPanel` 在 `onChange(output)` 时调用，缓存最近一次结果，避免高频 IPC。

## Technical Design

- Data model（Rust）：

  ```rust
  pub enum OutputTarget {
      Local(PathBuf),
      MountedRemote { kind: MountKind, path: PathBuf, fs_type: Option<String> },
      Remote(RemoteUri),
  }

  pub enum MountKind { Unc, Mounted }

  pub struct RemoteUri {
      pub scheme: String,
      pub host: String,
      pub port: Option<u16>,
      pub user: Option<String>,
      pub path: String,
      pub query: BTreeMap<String, String>,
  }
  ```

- API/commands：
  - 内部：`pub fn parse_output_target(raw: &str) -> OutputTarget`
  - IPC：`#[tauri::command] fn parse_output_target_command(raw: String) -> OutputTargetDescriptor`
- 文件结构（新增）：
  - `src-tauri/src/output_target/mod.rs`
  - `src-tauri/src/output_target/scheme.rs`（URL 解析，禁止依赖 `url` crate 的网络特性）
  - `src-tauri/src/output_target/unc.rs`
  - `src-tauri/src/output_target/mount.rs`（条件编译，`#[cfg(target_os = "...")]`）
- Storage：无持久化。
- Events：无。
- 依赖：`statfs` 用 `nix`（Linux/macOS）+ Windows 用 `windows-sys`；若引入风险大可降级为"按前缀认定为 MountedRemote::Mounted，不查 fs_type"，N3 同步放宽。

### Test cases (write failing tests first)

- [ ] Case 1: `parse_output_target("/Users/alice/build")` → `Local`
- [ ] Case 2: `parse_output_target("C:\\build\\out")` → `Local`（在所有平台都通过，含 Windows 盘符）
- [ ] Case 3: `parse_output_target("\\\\nas01\\releases\\app")` → `MountedRemote { kind: Unc }`
- [ ] Case 4: `parse_output_target("//nas01/releases/app")` → `MountedRemote { kind: Unc }`（POSIX 双斜杠也识别）
- [ ] Case 5: `parse_output_target("/Volumes/build-share/out")` 在 macOS 上 → `MountedRemote { kind: Mounted, fs_type: Some("smbfs"|"afpfs"|...) }`（用 fake `statfs` 注入）
- [ ] Case 6: `parse_output_target("/mnt/nas/out")` 在 Linux 上 + fake `statfs` 返回 `cifs` → `MountedRemote { kind: Mounted, fs_type: Some("cifs") }`
- [ ] Case 7: `parse_output_target("sftp://deploy@build.example.com:22/var/www")` → `Remote(RemoteUri { scheme: "sftp", host, port: 22, user, path })`
- [ ] Case 8: `parse_output_target("s3://bucket/prefix?region=ap-southeast-1")` → `Remote` 且 `query["region"] == "ap-southeast-1"`

- [ ] Case 10: 空字符串 → `Local(PathBuf::from(""))`
- [ ] Case 11: `parse_output_target_command` IPC 命令通过 `ts-rs` 生成的类型与 Rust 字段名一致
