## Context

Diagnostics bundles now exist, but teams still manually convert bundle content into issue descriptions.

## Goals

- Generate issue draft text from selected diagnostics bundle
- Include signature summary, impact, and reproduction references
- Support quick copy and markdown-friendly formatting

## Non-Goals

- Direct API posting to GitHub/Jira
- Template marketplace

## Delivered

- Added issue draft generator utility for failure groups
  - emits markdown structure with summary, representative command, recent occurrences, and follow-up checklist
  - includes snapshot references with output-dir fallback when snapshot is not exported
- Added unit tests for issue draft generation
  - validates signature/frequency output and snapshot fallback rendering
- Added one-click `复制 Issue 草稿` action in failure drill-down panel
  - builds draft from current selected failure group
  - reuses clipboard copy flow and toasts for consistent UX

## Acceptance Criteria

- [x] AC1: User can generate issue draft from a failure group
- [x] AC2: Draft includes signature, frequency, representative command, and snapshot references
- [x] AC3: Draft is copy-ready and markdown-formatted
